## Tabs y acordeones

Tabs y acordeones son dos patrones comunes para organizar y mostrar contenido de forma progresiva. Aunque visualmente similares, tienen patrones ARIA y comportamientos diferentes.

### 🗂️ Patrón de Tabs (Pestañas)

**Uso:** Cuando el usuario necesita ver diferentes vistas del mismo conjunto de información, pero solo una a la vez.

#### Estructura HTML/ARIA

```
<div class="tabs">
  <!-- Lista de pestañas -->
  <div role="tablist" aria-label="Información de perfil">
    <button
      role="tab"
      aria-selected="true"
      aria-controls="panel-personal"
      id="tab-personal"
      tabindex="0"
    >
      Datos personales
    </button>
    <button
      role="tab"
      aria-selected="false"
      aria-controls="panel-contacto"
      id="tab-contacto"
      tabindex="-1"
    >
      Contacto
    </button>
    <button
      role="tab"
      aria-selected="false"
      aria-controls="panel-seguridad"
      id="tab-seguridad"
      tabindex="-1"
    >
      Seguridad
    </button>
  </div>

  <!-- Paneles de contenido -->
  <div
    role="tabpanel"
    id="panel-personal"
    aria-labelledby="tab-personal"
    tabindex="0"
  >
    <h3>Datos personales</h3>
    <p>Contenido del panel...</p>
  </div>

  <div
    role="tabpanel"
    id="panel-contacto"
    aria-labelledby="tab-contacto"
    tabindex="0"
    hidden
  >
    <h3>Contacto</h3>
    <p>Contenido del panel...</p>
  </div>

  <div
    role="tabpanel"
    id="panel-seguridad"
    aria-labelledby="tab-seguridad"
    tabindex="0"
    hidden
  >
    <h3>Seguridad</h3>
    <p>Contenido del panel...</p>
  </div>
</div>
```

#### ⌨️ Comportamiento de teclado requerido

| Tecla | Acción |
| --- | --- |
| `Tab` | Entra al tablist (primer/último tab activo), luego al panel activo |
| `→` | Activa el siguiente tab (circular) |
| `←` | Activa el tab anterior (circular) |
| `Home` | Activa el primer tab |
| `End` | Activa el último tab |

#### 📝 JavaScript completo

```
class Tabs {
  constructor(container) {
    this.tablist = container.querySelector('[role="tablist"]');
    this.tabs = [...this.tablist.querySelectorAll('[role="tab"]')];
    this.panels = [...container.querySelectorAll('[role="tabpanel"]')];

    this.tabs.forEach((tab, index) => {
      tab.addEventListener('click', () => this.activateTab(index));
      tab.addEventListener('keydown', (e) => this.handleKeydown(e, index));
    });
  }

  activateTab(index) {
    // Desactivar todos los tabs
    this.tabs.forEach((tab, i) => {
      tab.setAttribute('aria-selected', i === index);
      tab.tabIndex = i === index ? 0 : -1;
    });

    // Mostrar/ocultar paneles
    this.panels.forEach((panel, i) => {
      panel.hidden = i !== index;
    });

    // Foco en el tab activado
    this.tabs[index].focus();
  }

  handleKeydown(e, currentIndex) {
    let newIndex;

    switch(e.key) {
      case 'ArrowRight':
        newIndex = (currentIndex + 1) % this.tabs.length;
        this.activateTab(newIndex);
        e.preventDefault();
        break;
      case 'ArrowLeft':
        newIndex = currentIndex === 0 ? this.tabs.length - 1 : currentIndex - 1;
        this.activateTab(newIndex);
        e.preventDefault();
        break;
      case 'Home':
        this.activateTab(0);
        e.preventDefault();
        break;
      case 'End':
        this.activateTab(this.tabs.length - 1);
        e.preventDefault();
        break;
    }
  }
}

// Inicializar
document.querySelectorAll('.tabs').forEach(el => new Tabs(el));
```

**🔑 Puntos clave:**

- Solo el tab activo tiene `tabindex="0"`
- Las flechas cambian de tab automáticamente (activación automática)
- Alternativamente, puedes usar activación manual: las flechas mueven el foco, Enter/Space activan

### 📋 Patrón de Acordeón

**Uso:** Cuando tienes múltiples secciones de contenido y quieres que el usuario pueda expandir/colapsar cada una independientemente.

#### Estructura HTML/ARIA

```
<div class="accordion">
  <!-- Sección 1 -->
  <h3>
    <button
      aria-expanded="false"
      aria-controls="content-1"
      id="accordion-header-1"
    >
      Pregunta frecuente 1
    </button>
  </h3>
  <div
    id="content-1"
    role="region"
    aria-labelledby="accordion-header-1"
    hidden
  >
    <p>Respuesta a la pregunta 1...</p>
  </div>

  <!-- Sección 2 -->
  <h3>
    <button
      aria-expanded="false"
      aria-controls="content-2"
      id="accordion-header-2"
    >
      Pregunta frecuente 2
    </button>
  </h3>
  <div
    id="content-2"
    role="region"
    aria-labelledby="accordion-header-2"
    hidden
  >
    <p>Respuesta a la pregunta 2...</p>
  </div>

  <!-- Sección 3 -->
  <h3>
    <button
      aria-expanded="true"
      aria-controls="content-3"
      id="accordion-header-3"
    >
      Pregunta frecuente 3
    </button>
  </h3>
  <div
    id="content-3"
    role="region"
    aria-labelledby="accordion-header-3"
  >
    <p>Respuesta a la pregunta 3...</p>
  </div>
</div>
```

#### ⌨️ Comportamiento de teclado

| Tecla | Acción |
| --- | --- |
| `Tab` | Navega entre botones del acordeón |
| `Enter` / `Space` | Expande/colapsa la sección |
| `↓` (opcional) | Mueve foco al siguiente botón |
| `↑` (opcional) | Mueve foco al botón anterior |
| `Home` (opcional) | Mueve foco al primer botón |
| `End` (opcional) | Mueve foco al último botón |

#### 📝 JavaScript

```
class Accordion {
  constructor(container, allowMultiple = true) {
    this.container = container;
    this.allowMultiple = allowMultiple;
    this.buttons = [...container.querySelectorAll('[aria-expanded]')];

    this.buttons.forEach((button, index) => {
      button.addEventListener('click', () => this.toggle(index));
      button.addEventListener('keydown', (e) => this.handleKeydown(e, index));
    });
  }

  toggle(index) {
    const button = this.buttons[index];
    const content = document.getElementById(button.getAttribute('aria-controls'));
    const isExpanded = button.getAttribute('aria-expanded') === 'true';

    // Si no permitimos múltiples, cerrar todos los demás
    if (!this.allowMultiple && !isExpanded) {
      this.buttons.forEach((btn, i) => {
        if (i !== index) {
          btn.setAttribute('aria-expanded', 'false');
          const otherContent = document.getElementById(btn.getAttribute('aria-controls'));
          otherContent.hidden = true;
        }
      });
    }

    // Alternar el actual
    button.setAttribute('aria-expanded', !isExpanded);
    content.hidden = isExpanded;
  }

  handleKeydown(e, currentIndex) {
    let newIndex;

    switch(e.key) {
      case 'ArrowDown':
        newIndex = (currentIndex + 1) % this.buttons.length;
        this.buttons[newIndex].focus();
        e.preventDefault();
        break;
      case 'ArrowUp':
        newIndex = currentIndex === 0 ? this.buttons.length - 1 : currentIndex - 1;
        this.buttons[newIndex].focus();
        e.preventDefault();
        break;
      case 'Home':
        this.buttons[0].focus();
        e.preventDefault();
        break;
      case 'End':
        this.buttons[this.buttons.length - 1].focus();
        e.preventDefault();
        break;
    }
  }
}

// Inicializar (allowMultiple: true permite múltiples secciones abiertas)
document.querySelectorAll('.accordion').forEach(el => {
  new Accordion(el, true);
});
```

**🔑 Puntos clave:**

- Puedes permitir múltiples secciones abiertas o solo una a la vez
- Usa encabezados (`<h3>`) para estructura semántica
- El `role="region"` es importante para que el lector identifique la sección

### Comparación: Tabs vs Acordeón

| Aspecto | Tabs | Acordeón |
| --- | --- | --- |
| **Navegación** | Flechas ← → entre tabs | Tab entre botones (flechas opcionales) |
| **Paneles visibles** | Solo uno a la vez | Uno o múltiples simultáneamente |
| **Gestión de foco** | tabindex en tabs (0 para activo, -1 para inactivos) | Todos los botones son enfocables naturalmente |
| **Roles ARIA** | tablist, tab, tabpanel | button, region (sin roles especiales de acordeón) |
| **aria-selected** | ✅ Sí (indica tab activo) | ❌ No se usa |
| **aria-expanded** | ❌ No se usa | ✅ Sí (indica estado expandido/colapsado) |
| **Uso típico** | Diferentes vistas del mismo contexto | FAQs, secciones independientes |

### Alternativa nativa: &lt;details&gt; y &lt;summary&gt;

Para acordeones simples, considera usar HTML nativo:

```
<details>
  <summary>Pregunta frecuente 1</summary>
  <p>Respuesta a la pregunta 1...</p>
</details>

<details>
  <summary>Pregunta frecuente 2</summary>
  <p>Respuesta a la pregunta 2...</p>
</details>

<details open>
  <summary>Pregunta frecuente 3</summary>
  <p>Respuesta a la pregunta 3...</p>
</details>
```

**✅ Ventajas:**

- Sin JavaScript necesario
- Accesibilidad integrada automáticamente
- Funciona sin CSS

**⚠️ Limitaciones:**

- Estilado más limitado (aunque mejorado en navegadores modernos)
- No soporta "solo uno abierto a la vez" sin JavaScript
- Animaciones más complejas de implementar

### Checklist de implementación

#### ✅ Tabs

- ☑️ Roles: tablist, tab, tabpanel
- ☑️ aria-selected en todos los tabs
- ☑️ aria-controls vincula tab con panel
- ☑️ aria-labelledby en paneles
- ☑️ tabindex: 0 en activo, -1 en inactivos
- ☑️ Flechas ← → funcionan
- ☑️ Home y End funcionan
- ☑️ Solo un panel visible a la vez

#### ✅ Acordeón

- ☑️ Botones con aria-expanded
- ☑️ aria-controls vincula botón con contenido
- ☑️ role="region" en contenido
- ☑️ aria-labelledby en regiones
- ☑️ Usa encabezados para estructura
- ☑️ Enter/Space funcionan
- ☑️ Estados se actualizan correctamente
- ☑️ Decide si permite múltiples abiertos

**💡 Consejo:** Para acordeones simples, considera usar `<details>` nativo. Para tabs o acordeones complejos con animaciones y comportamiento personalizado, implementa el patrón ARIA completo o usa una librería accesible.
